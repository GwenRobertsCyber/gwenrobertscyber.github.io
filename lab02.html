Lab 02: Advanced Secure Enclave Configuration

Time Estimate: 60 to 90 minutes

Objective

Configure an isolated virtual machine environment suitable for malware analysis or handling controlled unclassified information. This simulates the strict air gapped standards used by major aerospace and defense contractors.

Part 1: The Air Gap Network

When dealing with volatile files, your VM cannot have any bridge to your physical network. We need to create a custom Host Only network.

  • Open VMware Workstation and go to the Virtual Network Editor.
  • Create a new custom VMnet and select Host Only.
  • Crucial Step: Uncheck the box that says connect a host virtual adapter to this network. This severs the tie to your physical machine.
  • Disable local DHCP service to prevent automatic IP assignment.

Part 2: Hypervisor Hardening

Advanced malware can sometimes escape the virtual machine if the hypervisor tools are left open. We need to lock the borders.

  • Go into your VM settings before booting.
  • Navigate to the Options tab and select Guest Isolation.
  • Disable both Enable drag and drop and Enable copy and paste.
  • Ensure Shared Folders are completely disabled.

Part 3: The Pristine Baseline

Before you introduce any suspicious files, you must freeze the system state so you have a clean slate to return to.

  • Boot the VM and apply all necessary OS security updates.
  • Power the VM down.
  • Take a snapshot and name it Base Clean Image DO NOT DELETE.

Homework Challenge

Now that your single machine is isolated, deploy a second virtual machine running REMnux or Kali Linux on that exact same custom VMnet. Configure static IP addresses for both machines and use Wireshark on the Linux machine to prove they can communicate with each other but absolutely cannot reach the outside internet!
