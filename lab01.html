Lab 01: File Signature Analysis with WinHex

Time Estimate: 20 to 30 minutes

Objective

Learn how to identify the true nature of a file by examining its magic bytes (file header) rather than relying on the file extension.

The Scenario

During a mock investigation, you seized a suspect's flash drive. The suspect claimed it only contained system logs, and you found a file named system_log.txt. However, when you open it in a text editor, it is just a bunch of scrambled, unreadable characters. You suspect the file extension was intentionally changed to hide illicit data.

Part 1: The Hex Dump

Below is a simulated hexadecimal dump of the first few bytes of that suspicious file. Highlight and copy this hex data.

Offset(h) 00 01 02 03 04 05 06 07 08 09 0A 0B 0C 0D 0E 0F 00000000 89 50 4E 47 0D 0A 1A 0A 00 00 00 0D 49 48 44 52 00000010 00 00 03 20 00 00 01 C2 08 06 00 00 00 1E 3D D8 00000020 A2 00 00 20 00 49 44 41 54 78 5E EC DD 77 58 54

Part 2: The Investigation

  • Open your hex editor (like WinHex or an online tool like HexEd.it).
  • Look specifically at the first 4 to 8 bytes of the file (the "Magic Bytes").
  • Use a file signature database (like Gary Kessler's File Signature Table) to look up what file type corresponds to those starting hex values.

Homework Challenge

Once you identify the true file type, rename the file on your local machine with the correct extension. What kind of file was the suspect actually trying to hide from you?
